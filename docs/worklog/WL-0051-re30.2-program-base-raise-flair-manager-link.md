## Why

Pre-authorized mitigation (B0). The raise had to mirror o0td's PAIRING (the
Wave-4 raise-alone broke SAMIR; o0td's whole-map shift preserved it) -- by
shifting EVERY region by the same delta, the SAMIR heap-arena disjointness is

change) from the link (the size/symbol question), per the committee ruling.

## Frictions / lessons

- **`git checkout` on an UNCOMMITTED working-tree file reverts to HEAD, not to
  the agent's edit.** During the kernel.ld ASSERT mutation-proof, I mutated the
  literal then `git checkout os/milton/kernel.ld` -- which restored the
  *committed* 0x38000, wiping the Step-A agent's uncommitted 0x40000 edit. Had
  to re-apply by hand. LESSON: to mutation-test an uncommitted file, copy it
  aside (or `git stash`), not `git checkout`.
- The shell kernel `.bin` was ALREADY at exactly the 224-sector window
  (113,485 of 114,688 bytes, ~1.2 KiB free) BEFORE FLAIR -- so any FLAIR link
  forced a geometry bump. Measuring `size build/kernel_shell.elf` (not the
  padded `.bin`) gave the real headroom.
- The "almost certainly need escalation" warning for the link did NOT
  materialize: the full Manager set, only ever compiled in isolation, links
  with ZERO unresolved/duplicate symbols. The +0x8000 raise was sufficient
  (shell `_kernel_end=0x3bad0`, ~17.4 KiB under the 0x3FF00 guard).
